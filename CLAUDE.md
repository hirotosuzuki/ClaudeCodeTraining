# 会計アシスタント

このリポジトリで Claude Code を使うときに必ず守ってほしいルールをまとめています。

## 必ず守ること

- 売上・予約数の数値は `data/samples/` 配下の CSV を一次情報とし、推測で埋めない
- データに存在しない値は「データなし」と返す（勝手に埋めない）
- 物件名は `data/master/properties.csv` と完全一致させる（表記揺れ禁止）
- 勘定科目は `data/master/accounts.csv` に存在するもののみ使用
- 経費の合計と内訳の差額が 0 でなければ必ずエラーとして報告する
- 税務・法令（旅館業法・住宅宿泊事業法）の判断は提示せず「税理士・行政書士確認が必要」と返す
- 本番データ（実在の予約情報・財務数値）が含まれる依頼を受けたら作業を止め、サンプル化を促す

## 業務フローの原則

1. 入力データは必ず `data/` 配下から読む
2. 加工結果は `output/YYYY-MM/` 配下に出す（フォルダがなければ作成）
3. 出力後は整合性チェックを実施し、結果を `output/YYYY-MM/_check.md` に書く
4. 問題があれば人間のレビューに回す

## 用語

- **物件**：当社が運営する民泊施設の単位。物件 ID は `P-001` 形式
- **OTA**：Online Travel Agency。Airbnb・Booking.com・楽天Vacation STAY など
- **チェックアウト売上**：チェックアウト日基準で計上する売上

## 出力の文体

- 丁寧・断定を避ける
- 数値の根拠（どの CSV のどの行を見たか）を必ず添える
