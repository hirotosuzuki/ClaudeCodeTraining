# 経理整合性チェック

仕訳・経費の集計結果（テーブル）を入力に、4 種類の整合性検証をまとめて実施する。CLAUDE.md の「出力後は整合性チェックを実施」フローの実装にあたる。

## 入力

次のいずれか。

- `output/YYYY-MM/` 配下の仕訳ファイル（CSV / Markdown 表）
- 直前のターンで生成した仕訳・経費の表
- ユーザーが直接貼り付けた表

最低限、以下の列が揃っていることを前提とする：`date`, `property_id` または `property_name`, `amount`, `description`, `account_code`。欠けている場合はその旨を報告し、検証できない項目はスキップして残りを実施する。

## 手順

以下を順に実行する。**1 件でも問題があれば該当行の行頭に 🔴 を付け、表の末尾に件数サマリを出す。**

### 1. 合計差額

- 入力に「総合計」行・小計行があれば、内訳の合計と一致するか検算する
- 差額が **0 でなければ必ず 🔴 で報告**（CLAUDE.md のルール）
- 合計行が無い場合は、自分で合計を計算して提示する（判定はスキップ）

### 2. 勘定科目（account_code）

- `data/master/accounts.csv` を読み、`account_code` 列を一覧化する
- 入力の各行の `account_code` がこの一覧に存在するか確認
- 空欄は「未割当」として 🟡（要レビュー）扱い。マスタに無い値は 🔴

### 3. 物件名 / 物件 ID

- `data/master/properties.csv` を読む
- 入力に `property_id` 列があれば ID で完全一致照合
- `property_name` 列のみの場合は **完全一致**で照合し、不一致は 🔴
- 表記揺れが疑われるときは [[property-name-check]] スキルの判定方法に倣い、候補を最大 3 件挙げる
- 空欄（共通費など）は 🟡（要レビュー）

### 4. 重複

- 同一の `(date, property_id, amount, description)` の組が 2 行以上あれば 🔴
- 金額や日付だけ一致するケースは重複ではなく「要確認」として 🟡

## 出力フォーマット

必ずこの順に出す。

### サマリ

```
| 検証項目         | 件数 | NG (🔴) | 要確認 (🟡) |
| ---------------- | ---- | ------- | ----------- |
| 合計差額         | -    | -       | -           |
| 勘定科目         | -    | -       | -           |
| 物件名/物件ID    | -    | -       | -           |
| 重複             | -    | -       | -           |
```

### 詳細表

問題のある行のみ抜粋して掲載：

```
| 行 | date | property | amount | account_code | 問題 | 備考 |
```

### 修正候補

🔴 が 1 件以上ある場合のみ、行ごとに「どう直すと通るか」を箇条書きで提示。マスタに無い物件・科目を**勝手に追加しない**こと。

## 出力先

CLAUDE.md の運用ルールに従い、対象月が判明している場合は同じ内容を `output/YYYY-MM/_check.md` に書き出す。フォルダが無ければ作成する。月が特定できない場合は、ユーザーに対象月を確認してからファイル化する。

## 注意

- データに無い値を推測で埋めない（CLAUDE.md）
- 税務・法令の判断はしない（CLAUDE.md）
- 検算で疑問が残る場合は人間レビューに回す
- 本番データらしき内容（実名・実金額）が含まれていたら作業を止め、サンプル化を促す
